git commit            adds a commit

git branch            creates branches, a copy of your code so you can work on it without losing your main code

git checkout          choose what branch you want to be on, switch to it

git merge             merges two branches, collecting codes at the same place

git rebase            copy/move your commits to a new base

git add               add a file to the staging area

git init              create a new repository

git clone             create a local copy of a remote repository 

git status            show the state of staging area

git diff              show difference between files

git rm                remove file from repository

git mv                move or rename a file in a repository

git log               show commit history

git reset             undo changes you have made

git remote            create a list of used remote repositories

git fetch             get a remote repository into your local but without merging to your main code

git pull              get the changes made on a remote repository from a remote repository with merging

git push              upload your commit to a remote repository

git tag               create a permanent tag for a commit

git show              show info about your commit

git config            show options for git environment


STAGING AREA
Area between local code and repository. Useful because you can have several copies of your code, allows you to edit and add commits before pushing it online.
